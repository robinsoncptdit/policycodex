- **Finding -> REPO-15 (filed + FIXED this session, `S`, Week 5).** The boot exposed a data-loss footgun: `policycodex_site/env.py:get_db_path` defaulted to `base_dir/db.sqlite3` (`/app/db.sqlite3`, ephemeral) whenever `POLICYCODEX_DB_PATH` was empty. The `/data` volume is used only because `.env.example` documents the path and compose's `env_file` supplies it; an admin who copies `.env.example` and deletes/comments that line silently loses the DB on every recreate. (My raw `docker run` without the env var reproduced it — DB landed at `/app/db.sqlite3`.) **Fix (TDD):** `get_db_path` gained an injectable `data_dir: Path = Path("/data")` param — explicit env still wins; else when the data volume is mounted (`data_dir.is_dir()`) persist to `/data/db.sqlite3`; else (local non-container, no `/data`) fall back to repo-root sqlite, so local dev is unchanged. Three unit tests replace the old single default test (no-volume / volume-present / env-wins); the kept default test made hermetic via an injected missing `data_dir`.
