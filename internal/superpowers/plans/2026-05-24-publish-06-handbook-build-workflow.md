# PUBLISH-06 Handbook Build Workflow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Vendor the Astro handbook plus a GitHub Actions build workflow into `repo-template/` so a diocese policy repo builds its handbook from its own `policies/` on merge to `main` and uploads a GitHub Pages artifact.

**Architecture:** Copy the app repo's `handbook/` Astro project into `repo-template/handbook/` (sample content stripped) via a `sync-handbook.sh` script. Add `repo-template/.github/workflows/build-handbook.yml` that, on push to `main`, copies the repo's `policies/` into the handbook content dir, runs `npm ci && npm run build && npm run verify`, and uploads a Pages artifact. Python structural tests in the main suite guard the vendored layout and the workflow; a local Astro build smoke proves the copy mechanism end to end.

**Tech Stack:** Astro ^5 (Node 20, npm), GitHub Actions (`actions/upload-pages-artifact@v3`), Python 3.14 + pytest for structural tests, bash for the sync script. Test interpreter: `ai/venv/bin/python` (run via the parent's absolute path inside a worktree); Astro builds via npm inside the worktree.

**Design doc:** `internal/superpowers/specs/2026-05-24-publish-06-handbook-build-workflow-design.md`

### Task 1: Generalize `verify-build.mjs` to any diocese's policies

The current script hardcodes the four sample slugs. Rewrite it to require `dist/index.html` plus at least one non-trivial policy page (any `index.html` under `dist/policies/`, at any depth, so flat policies and bundles both count).

**Files:**
- Modify: `handbook/scripts/verify-build.mjs`

- [ ] **Step 1: Replace the script body with the generalized version**

```javascript
#!/usr/bin/env node
/**
 * Verify that `npm run build` produced a usable handbook:
 *   - dist/index.html exists and is non-trivial
 *   - at least one policy page exists (any non-trivial index.html under
 *     dist/policies/, at any depth - flat policies and bundles both count)
 * Generic across dioceses: no policy slugs are hardcoded.
 */
import { existsSync, readFileSync, readdirSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = dirname(fileURLToPath(import.meta.url));
const dist = join(__dirname, '..', 'dist');

const MIN_BYTES = 100;
let failed = 0;

function checkFile(rel) {
  const full = join(dist, rel);
  if (!existsSync(full)) {
    console.error(`MISSING: dist/${rel}`);
    return false;
  }
  if (readFileSync(full, 'utf-8').length < MIN_BYTES) {
    console.error(`TOO SMALL: dist/${rel}`);
    return false;
  }
  return true;
}

// 1. Home page.
if (checkFile('index.html')) {
  console.log('OK: dist/index.html');
} else {
  failed += 1;
}

// 2. At least one policy page anywhere under dist/policies/.
function* walkIndexHtml(dir) {
  if (!existsSync(dir)) return;
  for (const entry of readdirSync(dir, { withFileTypes: true })) {
    const full = join(dir, entry.name);
    if (entry.isDirectory()) {
      yield* walkIndexHtml(full);
    } else if (entry.name === 'index.html') {
      yield full;
    }
  }
}

let policyPages = 0;
for (const page of walkIndexHtml(join(dist, 'policies'))) {
  if (readFileSync(page, 'utf-8').length >= MIN_BYTES) {
    policyPages += 1;
  }
}
if (policyPages >= 1) {
  console.log(`OK: ${policyPages} policy page(s)`);
} else {
  console.error('No non-trivial policy pages found under dist/policies/');
  failed += 1;
}

if (failed > 0) {
  console.error(`\n${failed} check(s) failed.`);
  process.exit(1);
}
console.log('\nBuild verification passed.');
process.exit(0);
```

- [ ] **Step 1: Write the sync script**

```bash
#!/usr/bin/env bash
# Re-vendor the app repo's handbook/ into repo-template/handbook/.
# Excludes build dirs and sample content; leaves an empty, tracked
# content/policies dir. Run this whenever the app handbook/ changes
# (PUBLISH-02..05 will).
set -euo pipefail

HERE="$(cd "$(dirname "$0")" && pwd)"
SRC="$HERE/../handbook"
DST="$HERE/handbook"

rm -rf "$DST"
mkdir -p "$DST"

# Copy the tracked project files (skip node_modules/dist/.astro entirely).
for item in .gitignore astro.config.mjs package.json package-lock.json \
            README.md tsconfig.json scripts src; do
  cp -R "$SRC/$item" "$DST/"
done

# Strip sample policy content; keep the dir tracked for the build copy step.
rm -rf "${DST:?}/src/content/policies/"*
touch "$DST/src/content/policies/.gitkeep"

echo "Vendored handbook -> $DST"
```

### Task 4: The build workflow

**Files:**
- Create: `repo-template/.github/workflows/build-handbook.yml`

- [ ] **Step 1: Write the workflow**

```yaml
name: Build handbook

on:
  push:
    branches: [main]
    paths:
      - 'policies/**'
      - 'handbook/**'

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Check out the policy repo
        uses: actions/checkout@v4
      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: npm
          cache-dependency-path: handbook/package-lock.json
      - name: Stage policies into the handbook content dir
        run: |
          rm -rf handbook/src/content/policies/*
          cp -r policies/* handbook/src/content/policies/
      - name: Install dependencies
        working-directory: handbook
        run: npm ci
      - name: Build the handbook
        working-directory: handbook
        run: npm run build
      - name: Verify the build output
        working-directory: handbook
        run: npm run verify
      - name: Upload the Pages artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: handbook/dist
```
