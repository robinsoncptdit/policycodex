# APP-20 L1 Foundational-Policy UI Gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** On the `/catalog/` list, hide the ordinary edit affordance for foundational policies and show a banner directing the editor to the typed-table UI, while leaving gate transitions (Publish) untouched, so the app's first deletion-protection layer (L1) is visible and demoable.

**Architecture:** Pure template change to `core/templates/catalog.html` plus new tests in `core/tests/test_catalog.py`. No view change is needed: `core.views.catalog` already passes each `row.policy` (a `LogicalPolicy`) to the template, and `LogicalPolicy.foundational` is already populated by INGEST-07's `BundleAwarePolicyReader`. The catalog row gains a mutate-affordance block guarded by `{% if not row.policy.foundational %}`: non-foundational rows get an `Edit` link to `policy_edit`; foundational rows get a `<span class="foundational-gate">` banner instead. Any future per-row Delete affordance lives inside the same not-foundational branch, so it inherits the gate by construction. The Publish form stays outside the gate because a foundational edit still flows drafted -> reviewed -> published through its own PR.

**Tech Stack:** Django 5+ templates, pytest-django.

**Ticket reference:** `PolicyWonk-v0.1-Tickets.md` APP-20 line 102. Scope confirmed with Chuck 2026-05-24: catalog-only L1 gate (no detail view, no Delete feature; both are absent from the codebase today). The ticket's "detail views" clause is deferred to a separate follow-up ticket (see Out of Scope below). The "frontmatter recognition" half of the ticket is already satisfied by INGEST-07 (`LogicalPolicy.foundational` / `.provides`) and the existing `(foundational)` badge.

**BASE:** `main` at SHA `e852ab9`.

```python
def test_catalog_foundational_policy_still_publishable_when_reviewed(client, user):
    """L1 gates Edit/Delete, not gate transitions: a foundational policy in
    'reviewed' state must still expose the Publish button while the ordinary
    edit-form link stays hidden."""
    client.force_login(user)
    policies = [_stub_policy(
        slug="document-retention",
        kind="bundle",
        title="Document Retention",
        foundational=True,
        provides=("classifications",),
    )]
    open_prs = [{
        "pr_number": 5,
        "head_branch": "policycodex/draft-document-retention",
        "gate": "reviewed",
        "url": "https://example.com/p/5",
    }]
    with override_settings(
        POLICYCODEX_POLICY_REPO_URL="https://example.com/x.git",
        POLICYCODEX_WORKING_COPY_ROOT="/tmp",
    ):
        with patch("core.views.Path.exists", return_value=True):
            with patch("core.views.BundleAwarePolicyReader") as MockReader:
                MockReader.return_value.read.return_value = iter(policies)
                with patch("core.views.GitHubProvider") as MockProvider:
                    MockProvider.return_value.list_open_prs.return_value = open_prs
                    response = client.get("/catalog/")

    body = response.content.decode()
    # Publish affordance present despite foundational status.
    assert 'action="/policies/document-retention/publish/"' in body
    assert "Publish" in body
    # The ordinary edit-form link is still hidden, and the banner still shows.
    assert 'href="/policies/document-retention/edit/"' not in body
    assert "foundational-gate" in body
```
